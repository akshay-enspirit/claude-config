# Engineering
- YAGNI
- DRY
- Deep modules: hide complexity behind narrow interfaces
- Check existing patterns before introducing new ones

# Behavior
- Concise responses. No trailing summaries.
- No comments unless the WHY is non-obvious.
- No abstractions beyond what the task requires.
- No error handling for impossible scenarios.
- Never use em-dashes.

# Docs
- `docs/overview.md`: high-level project context only.
- Module docs (`docs/<module>.md`): specifics, decisions, key file paths, related doc paths.
- Concise over grammar; never ambiguous.
- Update the relevant doc whenever something worth noting changes.

# Git
- Commit messages: why, not what.
- Never commit or push without being asked.
